The CSS Cascade (Priority Order)

When multiple rules target the same property on the same element, the cascade determines the winner using these criteria in order:

  1. Origin & Importance: User-agent → Author → Author !important → User-agent !important
  2. Inline styles: style="" attribute beats all selectors (except !important in layers)
  3. @layer order: Unlayered rules beat all layers. Later layers beat earlier layers. !important reverses layer order.
  4. Specificity: (ID count, Class/Attr/Pseudo-class count, Element/Pseudo-element count)
  5. Source order: Last rule wins when everything else is equal

@layer and !important

This is the most confusing part of modern CSS cascade:

  • Normal rules: Unlayered > later layers > earlier layers
  • !important rules: Earlier layers > later layers > unlayered. The order reverses!
  • This means !important in an early @layer can beat !important in a later layer

Specificity Calculation

  • ID selectors (#id): 1-0-0
  • Class, attribute, pseudo-class (.class, [attr], :hover): 0-1-0
  • Element, pseudo-element (div, ::before): 0-0-1
  • :where() contributes 0 specificity. :is() and :not() use their most specific argument.
